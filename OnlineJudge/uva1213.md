# 根據你的程式碼分析：最終二維矩陣的結構

## DP矩陣的含義

你的程式碼建立了一個 **dp** 的二維陣列，其中：[^1]

- **dp[i][j]** 表示：用 **j** 個不同的質數相加得到 **i** 的方案數
- **i** 的範圍是 0~1120（目標和）
- **j** 的範圍是 0~14（使用的質數個數）

## 關鍵特徵分析

### 初始狀態

- **dp = 1**：用 0 個質數組成 0 的方法有 1 種（空集合）
- 其他位置初始為 0


### 重要模式觀察

1. **K=1 的情況**：
    - dp = 1（質數 2）[^2][^3]
    - dp = 1（質數 3）[^3][^4]
    - dp = 1（質數 5）[^5][^3]
    - 等等...每個質數對應一種方案
2. **K=2 的有趣案例**：
    - dp = 1（3+5=8）[^6][^2]
    - dp = 2（3+13=16, 5+11=16）[^7][^2]
    - dp = 3（5+19=24, 7+17=24, 11+13=24）[^8][^2]
3. **題目例子驗證**：
    - **dp = 2**，正好對應題目提到的 {2,3,19} 和 {2,5,17}[^4][^8]

## 程式碼核心邏輯

你的動態規劃使用了**01背包**的經典寫法：

```cpp
for (int k = 0; k < prime.size(); ++k) 
    for (int i = 1120; i >= prime[k]; --i)
        for (int j = 14; j >= 1; --j) 
            dp[i][j] += dp[i - prime[k]][j - 1];
```

- **從大到小遍歷**：確保每個質數只用一次
- **狀態轉移**：dp[i][j] += dp[i-prime[k]][j-1]
- **意義**：如果選擇 prime[k]，則目標和減少 prime[k]，使用的質數個數減少 1
